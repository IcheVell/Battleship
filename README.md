# Клиент-серверная игра «Морской бой» (Battleship)

Интерактивная консольная игра «Морской бой» с поддержкой многопользовательского режима, игры против искусственного интеллекта, режима наблюдателя и панели администратора. Проект реализован на основе клиент-серверной архитектуры с использованием протокола WebSocket для обеспечения двустороннего обмена данными в реальном времени.

## Основные возможности
* Одиночный режим (PvE): Матч против встроенного бота.
* Сетевой режим (PvP): Создание игровых сессий и игра с удаленными соперниками.
* Режим наблюдателя: Подключение к активным игровым сессиям для просмотра матча в реальном времени.
* Консольный интерфейс: Отображение игровых полей с использованием ANSI-графики, вывод системных логов и меню управления.
* Панель администратора: Постраничный просмотр истории завершенных матчей, пошаговый реплей игр, функции архивации и удаления игровых сессий.

---

## Технологический стек
* Серверная часть: Java, Spring Boot, Spring Data JPA, WebSockets.
* База данных: PostgreSQL.
* Клиентская часть: Java (Core), Jackson (сериализация/десериализация JSON), WebSocket Client.
* Инфраструктура: Docker, Docker Compose.

## Инструкция по запуску сервера через Docker

Серверное приложение и база данных PostgreSQL изолированы в Docker-контейнерах для упрощения процесса развертывания.

### 1. Настройка переменных окружения (.env)
В корневой директории серверной части проекта (в той же папке, где находится файл `docker-compose.yml`) необходимо создать файл `.env` со следующими параметрами конфигурации:

```env
# Параметры подключения к базе данных PostgreSQL
POSTGRES_DB=battleship_db
POSTGRES_USER=postgres
POSTGRES_PASSWORD=secretpassword
DB_PORT=5432

# Параметры конфигурации сервера Spring Boot
SERVER_PORT=8080
```

### 2. Запуск контейнеров
Для сборки образов и запуска контейнеров откройте терминал в папке проекта и выполните следующую команду:

```bash
docker-compose up -d --build
```

## Инструкция по запуску клиента через скрипт run.sh

Клиентское приложение запускается локально. Для работы программы требуется установленная среда выполнения Java (JDK версии 17 или выше).

### 1. Настройка прав доступа (для операционных систем Linux и macOS)
Перед первым запуском необходимо предоставить скрипту права на выполнение. Откройте терминал в директории клиента и выполните:
```bash
chmod +x run.sh
```

### 2. Запуск приложения
Для старта клиентской части выполните команду:
```bash
./run.sh
```

## Управление и игровые команды

После подключения к серверу и авторизации под уникальным именем пользователя, взаимодействие с системой происходит посредством текстовых команд.

### Главное меню
* `1` — Инициализировать игру против бота.
* `2` — Создать сетевую игровую комнату для ожидания второго игрока.
* `3` — Вывести список доступных для подключения комнат.
* `admin` — Переход в панель администратора (команда доступна только из главного меню).

### Фаза расстановки кораблей
* `placeauto` — Автоматическая генерация и валидация расстановки флота из 21 корабля.
* `addship <размер> <координата_начала> <ориентация>` — Ручное добавление корабля на поле.
  * Пример: `addship 4 B5 h` (четырехпалубный корабль, начиная с клетки B5, горизонтально).
* `submitships` — Подтверждение готовности расстановки и переход к фазе боя.

### Фаза ведения боя
* `fire <координата>` — Произвести выстрел по указанной клетке поля соперника.
  * Пример: `fire A15` или `fire G7`.

### Системные команды
* `back` — Возврат на предыдущий экран, выход из лобби или фиксация поражения (если матч уже начался).
* `exit` — Завершение работы клиентского приложения.

## Функционал панели администратора

Для перехода в режим администрирования введите команду `admin` на стартовом экране приложения. На экране отобразится таблица завершенных игровых сессий с поддержкой постраничной навигации.

### Команды управления:
* `next` / `prev` — Переключение страниц списка игровых сессий.
* `watch <ID>` — Запуск пошагового реплея выбранного матча. Переход к анализу следующего хода осуществляется нажатием клавиши `Enter`.
* `archive <ID>` — Принудительное завершение активной сессии с переводом ее в архивный статус (игроки будут автоматически отключены и возвращены в главное меню).
* `delete <ID>` — Полное удаление записи об игровой сессии и связанных ходах из базы данных.
* `back` — Возврат в главное меню приложения.
